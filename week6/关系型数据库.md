# 一、基础概念

## 1.1 什么是关系型数据库

- 关系型数据库就是用表格来组织数据的仓库，每一个表格都有行（记录）和列（字段）

- 不同的表格通过关系（主键、外键）连接起来
  
  ## 1.2 核心概念

- 主键：唯一标识一行数据的列(意味着每一行的值必须唯一且不为空）

- 外键：指向另一个表主键的列

- 关系类型：一对多、一对一、多对多

# 二、创建数据库

## 2.1 选择数据库管理系统

- 通常选择SQLite,常见的还有MYSQL、PostgreSQL
  
  ## 2.2 连接数据库或创建文件
  
  ```python
  conn = sqlite3.connect('library.db') # 如果括号内文件不存在，自动创建
  cursor = conn.cursor()  # 获取游标
  ```

- 执行SQL语句：将你写的SQL指令发给数据库去执行

- 游标：有些SQL语句会返回一个结果集（多行多列的excel表），你需要逐行地处理这些数据，游标就是执行结果集中某一行的指针或迭代器
  
  ## 2.3 创建表
  
  ```python
  cursor.execute('''
  CREATE TABLE IF NOT EXISTS authors( # authors是表的名字，如果authors这张表已经存在，则什么都不做
    author_id INTEGER PRIMARY KEY,  
    name TEXT NOT NULL,
    nationality TEXT
  );
  ''')
  ```

- execute()作用是把括号里的SQL语句发送给数据库执行

- 上述代码具体含义

| 列名          | 数据类型    | 约束          | 含义             |
| ----------- | ------- | ----------- | -------------- |
| author_id   | INTEGER | PRIMARY KEY | 作者id,整数类型，作为主键 |
| name        | TEXT    | NOT NULL    | 作者姓名，文本类型，不能为空 |
| nationality | TEXT    | (无）         | 国籍，文本类型，可以为空   |

## 2.4 提交更改

```python
conn.commit() # 只要对数据库的数据或结构产生了更改，就需要提交
```

# 三、对数据库进行增删改查

## 3.1 插入数据

```python
cursor.execute("INSERT INTO authors(name,nationality) VALUES(?,?)",
                ("j.k.Rowling","British"))
# 插入后需要commit()才会真正写入文件
```

## 3.2 查询数据

```python
# 查询所有书籍及其作者（通过JOIN关联表authors和books）
cursor.execute('''
SELECT books.title , authors.name
FROM books
JOIN authors ON books.author_id = authors.author_id;
''')
results = cursor.fetchall() # 获取所有结果行
# 此时results返回的是[('西游记','吴承恩'),('红楼梦','曹雪芹')]
for book,author in results:
    print(f"书籍：{book},作者：{author}")
# 此时输出的是 书籍：西游记，作者：吴承恩
#            书籍：红楼梦，作者：曹雪芹
```

- FROM books：意思是主表是books

- JOIN用来将books和authors两个表按照某种关系合并成一张临时表

- ON后面指定的是连接条件

- books.author_id = authors.author_id：只有当books表中的author_id值等于authors表中的author_id值时，这两条记录才会匹配到一起，目的是将books中的书名和authors中的作者名通过author_id对应起来

- fetchall()：获取执行查询后返回的所有结果行

## 3.3 更新数据

```python
cursor.execute("UPDATE books SET published_year = ? WHERE title = ?",
(1998,'Harry Potter'))
# 记得commit()
```

- UPDATE...SET...WHERE：更新...表，将...设置为...，当...为...时
  
  - 更新books表，将published_year列设置为某个值，只有当title等于某个值的时候

## 3.4 删除数据

```python
cursor.execute("DELETE FROM borrowers WHERE name = ?",('Bob',))
# 记得commit()
```

## 3.5 关闭连接

```python
conn.close()
```

# 四、数据库范式

> 范式是关系型数据库设计的一种规范

## 4.1 第一范式（1NF）

- 要求：每个列都必须是原子性的，不可再分
  
  - 每个列只存储一个值

## 4.2 第二范式(2NF)

- 要求：满足1NF+没有部分依赖
  
  - 部分依赖：当主键是复合主键时，非主键列不能只依赖于主键的一部分

## 4.3 第三范式（3NF）

- 要求：满足2NF+没有传递依赖
  
  - 传递依赖：非主键列不能依赖其他非主键列

## 4.4 BCNF

- 要求：比3NF更严格，要求每一个决定因素都包含候选键
